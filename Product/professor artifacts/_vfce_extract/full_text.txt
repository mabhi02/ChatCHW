===== PAGE 2 =====
Current digitization is a manual, error-prone translation 
task
●
The Status Quo: Developers interview MDs or interpret Ministry of Health 
(MOH) PDFs to write code (CHT, ODK, OpenSRP).
●
The Risk: "Trust-based" transcription introduces silent errors:
○
Missed exclusion criteria (e.g., age cutoffs).
○
Unsafe defaults (e.g., treating missing temperature as "normal").
○
Logic gaps (e.g., stockout scenarios ignored).
●
Code takes years to write!  So few CHWs have clinical support on their 
phones

===== PAGE 3 =====
The Solution: Zero-Trust Engineering
Core Philosophy: "Zero-Trust."  We assume every artifact is flawed until 
proven otherwise
The Verification Stack:
●
Adversarial Extraction: "Red Teams" hunting for logic gaps at each step of creating 
the clinical logic and code
●
Formal Verification: Mathematical proofs of desirable properties: no infinite loops, 
decision trees have graceful endpoints, etc. 
●
Human Sovereignty: Mandatory sign-offs of clinical logic & of translations
●
Traceability: Cryptographic links to source text
●
Differential Testing: Synthetic patient simulations show approved clinical logic 
operates the same as the final code

===== PAGE 17 =====
What inputs does each DMN require? 
A DMN requires some boolean inputs.  
Each boolean input requires (and implies) 
1. A yes/no question or exam (e.g., Is there blood in stool?)
2. A yes/no data from the medical record (not yet implemented, and 
not a priority)
3. boolean, Categorical or continuous data from question, exam or 
medical record + some logic to create yes/no.  (e.g., "Ask age in 
months or years") converts to age in months converts to 
age_le_12_mo or whatever.
So the one artifact created by the LLM is to draw out these 
implications and format them so the compiler can output the 
appropriate XLSForm.
For example: a cough diagnosis table might have inputs:  
p_age_2_59m
p_fast_breathing
p_chest_indrawing
p_danger_sign
So implies logic
* p_age_2_59m = true if age_mo >= 2 & age_mo <=59 
* p_fast_breathing = F( measurement   of respiration rates / minute, age)
* p_danger_sign = true if any of a bunch of earlier observations or questions on 
convulsions, etc. are true
In turn, that logic implies data collection (if not already collected): 
●
ask or find in medical record: age_mo 
●
measurement of respiration rate
●
observation of chest indrawing
●
observation of all the general danger signs
One intermediate artifact is the source of each variable:
●
Some data are from the opening module: age_mo and observation of 
all the general danger signs.  So no need to collect in later modules. 
●
Some data may be collected only in one module, so always collect for 
that DMN
●
Some data may be already known if they went to a particular earlier 
module.  So need to check "If temp_C = missing then ex_temp_C" 
where ex_temp_C is instructions to the CHW to take temperature.

===== PAGE 40 =====
Traceability and Replicability
Traceability: Every rule and line of code is linked to a specific page and 
text snippet from the source manual(s)
Replicability: We cryptographically sign and freeze the clinical logic after 
MOH approval. All code and libraries that validate the clinical logic and 
then compile the logic into code such as XLSForm live in a single 
container with a unique hash.  
Everything after clinical sign-off is deterministic (no AI), so is bit-by-bit 
replicable
