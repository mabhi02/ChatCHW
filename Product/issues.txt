  What the new smoke test should look like

  The most important thing the smoke test must answer is: does the OpenAI Responses API actually accept the parameters vision.py builds. Two specific
  findings (B6 detail=original, B7 text=verbosity) are model-and-version-specific and can ONLY be verified by hitting the live API. Static analysis is
  insufficient because the OpenAI Python SDK's TypedDicts may have been updated post my training cutoff but Agent A's training cutoff is the same, so we
  can't trust the static check alone.

  ★ Insight ─────────────────────────────────────
  - This is a case where the smoke test isn't validating what works — it's actively trying to falsify two specific bug claims. If the API returns 200 on
  these exact parameters, Agent A was wrong and we save effort. If it returns 400, we have empirical proof and a clear fix.
  - The cheapest way: synthesize the call with a tiny test image, NOT a real PDF. Skip Unstructured entirely. Just verify the OpenAI parameter contract.
  - A SECOND smoke test (after the parameter check) should exercise the full vision.py path on a single real cropped table, to verify the integration
  end-to-end.
  ─────────────────────────────────────────────────

  The proposed smoke test structure:

  PART A — Falsify/confirm B6 + B7 (the OpenAI parameter contract questions)
    1. Build a tiny test PNG (~100 KB, plain text "Hello world")
    2. Call OpenAI Responses API with the EXACT kwargs vision.py builds at lines 843-849:
         model="gpt-5.4"
         text={"verbosity": "high", "format": {...}}
         input=[{type: "input_image", image_url: ..., detail: "original"}]
         max_completion_tokens=300
    3. Capture the HTTP status + error body
    4. If 400 with "Invalid value 'original'" -> B6 confirmed, fix to "high"
    5. If 400 with "Unrecognized key 'verbosity'" -> B7 confirmed, remove key
    6. If 200 -> both findings discounted, vision.py is fine

  PART B — Verify the full vision.py code path on a real crop
    Only run if Part A returns 200. Otherwise fix the bugs first.
    1. Slice 5 pages from CHW Navigator PDF
    2. Send to Unstructured hi_res via the existing unstructured_client module
    3. Call vision.py's _refine_table on the first detected Table element
    4. Verify the returned TableRefinement passes Pydantic validation
    5. Print elapsed time + result summary

  PART C — Verify B5 (IngestionJobState) by parsing a real Redis state write
    1. Trigger pipeline.update_job_state("test-id", "running")
    2. Read the value back from Redis
    3. Try to parse as IngestionJobState
    4. If ValidationError -> B5 confirmed, started_at needs to be Optional or always set

  Total cost: ~1 image generation + 1 vision call (~$0.005) + ~5 pages of Unstructured quota (~free tier) + 1 Redis write/read

  Total wall-clock: ~30-60 seconds

  This smoke test is falsification-driven: we run it specifically to verify the highest-risk audit findings. It does NOT validate the cache, the assembler,
  the RLM runner, or the FastAPI integration — those bugs are static-analysis-confirmable and don't need runtime testing.
